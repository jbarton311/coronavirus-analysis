# Data Preview

## Caveats
- This dataset relies on data sourced from JHU found [here](https://github.com/CSSEGISandData/COVID-19)
    - As this is being done in my spare time, I do not guarantee the accuracy. 
- I strongly encourage users QA their own data/reporting before use.
  - Please let me know if any issues are discovered.

#### Known Issues
There are known issues with US recovery data at the moment. Hopefully these will be fixed soon.
        
## Overview of Dataset
- `HOPKINS_CLEANED.csv` is a daily dataset that combines Coronavirus metrics for cases, recoveries, and deaths.
    - This data contains `new daily metrics` as well as `running totals`.
- Rows will be unique on `province_or_state`, `country_or_region` and `date`

## Data Clean Up    
#### Formatting of the data has been done on the data that includes:
- Converting data structure to more of a stacked, daily dataset
- Aggregating all US data to the state level
    - Some historical US data was reported at the city/county level

#### More minor changes
- Aggregating `DC` and `District of Columbia` into `DC`
- Changing `Korea, South` to `South Korea`
- Several new fields have been added to the JHU dataset for more efficient analysis in BI tools
- Column names have been slightly modified/formatted

#### Column Explanations
- `first_case_state_rank` and `first_case_country_rank` are rank columns with the first day starting when total cases went over `5` at the state and country level respectively. 
  - Starting at `5` instead of `1` gives a better sense of when outbreaks begin at the state level
- The `hundred_case` (`hundred_case_state_rank` and `hundred_case_country_rank`) columns are the same style of rank columns
  - These fields start counting at `100` total cases.
